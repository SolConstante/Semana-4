## Descripción
Este proyecto consiste en un sistema básico de gestión de restaurante desarrollado en Python utilizando Programación Orientada a Objetos. Permite registrar productos y clientes, organizando el código en módulos para mantener una estructura clara y ordenada. Esta organización facilita la comprensión, el mantenimiento y la futura mejora del programa.

## Estructura del proyecto
restaurante_app/
│
├── modelos/
│   ├── producto.py
│   └── cliente.py
│
├── servicios/
│   └── restaurante.py
│
└── main.py

## Componentes

### Producto
Representa los productos disponibles en el restaurante.

### Cliente
Representa a los clientes registrados.

### Restaurante
Gestiona los productos y clientes del sistema.

### main.py
Archivo principal encargado de crear los objetos y ejecutar el programa.

## Reflexión

La modularización del software ayuda a mantener el código más ordenado y fácil de entender. Al dividir el programa en diferentes módulos y asignar una responsabilidad específica a cada clase, resulta más sencillo realizar cambios, corregir errores y reutilizar partes del código en otros proyectos. Además, esta forma de organización permite desarrollar programas más claros, prácticos y fáciles de mantener a medida que crecen.
